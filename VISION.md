# Senlo — Product Vision

## Суть

Senlo — это open-source платформа для transactional email с облачным managed-вариантом.

Модель: **open-source ядро + cloud SaaS** (как Plausible, Umami, Ghost).

- Open-source = маркетинг, доверие, сообщество
- Cloud = монетизация

---

## Проблема

Существующие решения плохо решают одну из двух вещей:

- **Listmonk** — мощный, но требует DevOps-экспертизы. Сложная установка, PostgreSQL, Go-шаблоны. Не для всех.
- **Klaviyo / Mailchimp** — дорогие, перегруженные фичами, vendor lock-in.
- **Postmark / Resend** — только доставка, без редактора шаблонов и логики.

Нет инструмента, который сочетает **простой UX + self-hosted контроль + разумную цену**.

---

## Решение

Senlo закрывает этот gap:

- Визуальный редактор шаблонов на базе MJML — без знания кода
- Self-hosted вариант для тех, кто хочет контроль над данными
- Cloud вариант для тех, кто не хочет заниматься DevOps
- Transactional-фокус: API-триггеры, вебхуки, логи доставки

---

## Целевая аудитория

**Self-hosted версия:**
- Разработчики и небольшие команды
- Компании с требованиями к приватности данных (GDPR, fintech, healthtech)
- Те, кто уже знает listmonk, но хочет лучший UX

**Cloud версия:**
- Инди-разработчики и небольшие SaaS-продукты
- Агентства, которым нужен white-label email для клиентов
- Shopify-мерчанты (как один из коннекторов, не основной фокус)

---

## Ключевое преимущество

**MJML-редактор** — это главный дифференциатор от listmonk.

Listmonk требует писать Go-шаблоны вручную. Senlo даёт визуальный редактор с адаптивными письмами из коробки. Это снижает порог входа с "разработчик" до "любой человек".

## Роадмап

### Фаза 1 — Open-source ядро
- [ ] Transactional email API
- [ ] MJML-редактор шаблонов
- [ ] Docker-образ, простая документация
- [ ] GitHub релиз

### Фаза 2 — Cloud
- [ ] Managed hosting (регистрация → сразу работает)
- [ ] Биллинг по объёму писем
- [ ] Дашборд: логи, статистика доставки, bounces
- [ ] **User Activity Log**: поиск по email для просмотра всей истории писем конкретного пользователя (поддержка и дебаг)

### Фаза 3 — Экосистема
- [ ] Коннекторы: Shopify, Stripe, любой вебхук
- [ ] White-label для агентств
- [ ] API-документация, SDK

---

## Цель

**$1–2k MRR** в первый год через cloud-версию.

Путь: open-source репо привлекает разработчиков → часть конвертируется в cloud → word of mouth внутри инди-сообщества.
